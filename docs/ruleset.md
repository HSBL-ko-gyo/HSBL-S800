# 爆速麻雀 ルールセット

## 参照方針

本PJは初心者向けの一人用麻雀練習アプリであり、完全な競技麻雀シミュレータではない。  
基本的なリーチ麻雀ルールを基準にしつつ、教材目的のため未対応・簡略化仕様を明示する。

麻雀ルールに関係する変更では、実装者の記憶だけに依存せず、この文書、`docs/rule-cases.md`、`src/rules/ruleRegistry.ts`、固定局面テスト、`validateGameState`、プレイログを照合する。

## 暫定仕様

- カンは現時点では宣言できない。
- 死牌14枚は `deadWall` として保持するが、通常進行ではまだ使用しない。
- 通常ツモは従来通り `wall` から行う。
- 下家・対面の捨て牌で発生した鳴き候補を、人間のツモ前境界まで `pendingReactionEvents` に蓄積する。これは通常麻雀ルールではなく、爆速麻雀の高速進行UIのための独自仕様。
- リーチ中はツモ切りのみ。
- リーチ宣言ボタンは、門前・自分の打牌番など基本条件を満たすなら有効にする。リーチできない打牌を選んだ場合は不成立として返す。
- フリテン判定はまず人間プレイヤーのロン可否に適用する。敵AI側のフリテン判定は未対応。
- 爆速麻雀の `pendingReactionEvents` 蓄積仕様により、一時フリテンの発生タイミングは通常麻雀の逐次確認と完全には一致しない場合がある。
- 初心者向け教材のため、一部ルールは段階的に導入する。
- 未対応ルールはバグではなく、明示された制限として扱う。

## フリテン仕様

### 捨て牌フリテン

現在の待ち牌のいずれかが自分の河にある場合、他家の捨て牌ではロンできない。ツモは可能。

例: 待ちが `3p / 6p` で、自分の河に `3p` がある場合、他家が `6p` を捨ててもロンできない。

### 一時フリテン

ロン可能な他家の捨て牌を見送った場合、次の自分のツモ番開始までロンできない。次の自分のツモ番が始まったら解除する。

爆速麻雀では、下家・対面の捨て牌で発生したロン候補を `pendingReactionEvents` に蓄積し、人間のツモ前境界でまとめて確認する。そのため、同一review内に複数のロン候補がある場合は、ロン可能イベントを見送った時点で、そのreview以降のロンを不可として扱う。

### リーチ後見逃しフリテン

リーチ後にロン可能な他家の捨て牌を見送った場合、その局の終了までロンできない。ツモは可能。

リーチ後に自分で和了牌をツモして自動ツモ切りした場合の扱いは、今後の追加検討対象とする。今回の実装では、他家の捨て牌に対するロン見逃しを対象にする。

### フリテン中のツモ和了

フリテンはロン制限であり、ツモ和了は可能。現行のツモ和了UI仕様範囲ではリーチ中のツモ和了を確認対象とする。非リーチ時のツモ和了UI拡張は別タスク。

## ルール変更時の必須手順

1. `docs/ruleset.md` を更新する。
2. `docs/rule-cases.md` にルールケースを追加または更新する。
3. `src/rules/ruleRegistry.ts` にルールIDを追加または更新する。
4. `validateGameState` の検査を追加または更新する。
5. 固定局面テストを追加または更新する。
6. 実装する。
7. `npm test -- --run` を実行する。
8. `npm run build` を実行する。
9. 実装者が最低1局プレイテストする。
10. ルール自己監査レポートを添える。

報告には対象ブランチ、対象コミット、固定seedを使った場合のseed、再現手順を含める。
